# FastAPI-based Attendance System for Year 11 Athena
#
# Stack Decision: FastAPI + Jinja2 + SQLite + AIO (async)
# - FastAPI: async, auto-validation, OpenAPI docs, fastest Python web framework
# - Jinja2: server-side rendering (no SPA complexity, direct CSRF, no CORS)
# - SQLite + aiosqlite: zero-config async DB, perfect for 20-student single-class
# - Field-level AES-256-GCM via cryptography: DB at-rest encryption
# - bcrypt: battle-tested password hashing (not scrypt/argon2 for availability on all platforms)
# - itsdangerous: signed session cookies (server-validated, tamper-proof)
#
# Rationale: A SPA (React/Vue) would add 30%+ complexity for zero benefit here.
# Server-rendered Jinja2 gives faster dev, simpler deployment, better SEO for dashboards,
# and natural CSRF protection. SQLite over PostgreSQL: no daemon, no config, encrypted at field level,
# and the data volume (~20 students) will never outgrow SQLite's capabilities.
